# Query Explanations & Samples
This document covers the full query catalog currently generated by `UsersQueriesRepository` in `src/queries_repository.rs`, including core queries, optional algorithm queries, and phase-1 additions.

## Dataset assumptions
- Primary label: `:User`
- Primary relationship: `:Friend`
- Common properties used by queries: `id`, `age`, and `bench_capacity`

## Complete query catalog (alphabetical by query id)
- `aggregate_age`: average age over all users.
- `aggregate_age_distinct`: count distinct age values.
- `aggregate_age_filtered`: average age for users `>= 18`.
- `aggregate_age_min_max_avg`: min/max/avg age in one query.
- `aggregate_count_users`: total user count (vendor-optimized in Falkor via `db.meta.stats()`).
- `aggregate_expansion_1`: 1-hop expansion.
- `aggregate_expansion_1_with_filter`: 1-hop expansion with `age >= 18`.
- `aggregate_expansion_2`: 2-hop expansion with `DISTINCT`.
- `aggregate_expansion_2_with_filter`: 2-hop expansion with `age >= 18`.
- `aggregate_expansion_3`: 3-hop expansion with `DISTINCT`.
- `aggregate_expansion_3_with_filter`: 3-hop expansion with `age >= 18`.
- `aggregate_expansion_4`: 4-hop expansion with `DISTINCT`.
- `aggregate_expansion_4_with_filter`: 4-hop expansion with `age >= 18`.
- `algo_harmonic_summary`: harmonic-centrality summary stats.
- `algo_max_flow_single_pair`: max-flow between two users using `bench_capacity`.
- `algo_msf_summary`: spanning-forest style edge/weight summary.
- `algo_pagerank_summary`: page-rank sample score.
- `all_shortest_paths_len`: `allShortestPaths` operator coverage.
- `call_subquery`: correlated `CALL { ... }` subquery.
- `count_friend_edges_plain`: plain edge count (optimizer reduction target).
- `count_users_plain`: plain node count (optimizer reduction target).
- `detach_delete_user`: `DETACH DELETE` clause coverage.
- `entity_path_introspection`: entity/path functions (`labels`, `type`, `properties`, `nodes`, `relationships`, `length`).
- `exact_5_hop_traverse_count`: fixed 5-hop traversal.
- `exact_6_hop_traverse_count`: fixed 6-hop traversal.
- `foreach_loop_mutation`: `FOREACH` mutation-loop coverage.
- `id_range_scan`: internal-id range scan.
- `id_seek`: internal-id point lookup.
- `indexed_in_list_predicate`: IN-list index utilization shape.
- `indexed_or_predicate`: OR-based index utilization shape.
- `merge_friend_edge_upsert`: MERGE on `Friend` relationship pattern.
- `merge_user_insert_path`: MERGE create-path (`ON CREATE`).
- `merge_user_upsert_existing`: MERGE match-path (`ON MATCH` updates).
- `neighbours_2`: 2-hop neighbor ids.
- `neighbours_2_with_data`: 2-hop neighbors returning full node.
- `neighbours_2_with_data_and_filter`: 2-hop neighbors with data + age filter.
- `neighbours_2_with_filter`: 2-hop neighbors with age filter.
- `optional_friend`: optional expansion from a seed user.
- `order_by_age`: full sort by `age`, `id`.
- `pattern_cycle`: 3-node cycle pattern.
- `pattern_long`: longer fixed pattern expansion.
- `pattern_short`: shorter fixed pattern expansion.
- `remove_user_property_and_label`: `REMOVE` on property + label.
- `shortest_path`: shortest path length between source and target.
- `shortest_path_with_filter`: shortest path length with non-empty-path filter.
- `single_edge_update`: update one existing `Friend` edge (`color` property).
- `single_edge_write`: create a `Friend` edge between two users.
- `single_vertex_read`: point lookup by user id.
- `single_vertex_update`: update `rpc_social_credit` on a user.
- `single_vertex_write`: create one `User` node.
- `union_all_ids`: `UNION ALL` composition.
- `union_distinct_ids`: `UNION` (distinct) composition.
- `unwind_rows`: row-level list fan-out with `UNWIND`.
- `value_join`: value-based join on `age`.
- `value_join_cnt`: count variant of the value join.
- `var_len_friends`: variable-length traversal (`*1..2`).
- `var_len_with_edge_where_filter`: variable-length traversal with edge filtering.
- `vertex_on_label_property`: lookup by label + property.
- `vertex_on_label_property_index`: same shape for index-path benchmarking.
- `vertex_on_property`: lookup by property without label predicate.

## Representative sample snippets
```cypher
MERGE (u:User {id: $id})
ON CREATE SET u.created_at = timestamp(), u.age = $age
RETURN u.id
```

```cypher
MATCH (u:User {id: $id})
FOREACH (x IN [1,2,3] | SET u.loop_counter = x)
RETURN u.loop_counter
```

```cypher
MATCH (s:User {id: $id})-[:Friend*6..6]->(t:User)
RETURN count(t) AS cnt
```

```cypher
MATCH p=(a:User {id: $id})-[r:Friend]->(b:User)
RETURN labels(a), type(r), properties(a), nodes(p), relationships(p), length(p)
LIMIT 1
```

## Vendor-specific notes
- `shortest_path`, `shortest_path_with_filter`, and `all_shortest_paths_len` have vendor-specific query text.
- `aggregate_count_users` uses a Falkor-optimized `db.meta.stats()` path.
- `algo_max_flow_single_pair` in Falkor uses one relationship type in config:
  - gets a single value from `db.relationshipTypes()`
  - passes `relationshipTypes: [relationshipType]`
